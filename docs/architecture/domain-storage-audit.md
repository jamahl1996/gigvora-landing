# Domain Storage Audit

Companion to [`mem://tech/no-domain-code-in-supabase`](../../.lovable/memory/tech/no-domain-code-in-supabase.md).

This audit confirms every enterprise Gigvora domain stores its code in the
user's source tree under `apps/`, `packages/`, `src/`, or `docs/` — never
inside the Lovable Cloud / Supabase project. Supabase is reserved for the
unrelated DailyMint habit-tracker demo only.

## Supabase project (Lovable Cloud) — what IS there

| Object | Purpose | Domain owner |
|---|---|---|
| `public.habits`, `public.habit_logs`, `public.profiles` | DailyMint habit tracker | DailyMint demo only |
| `public.handle_new_user()` trigger function | Backfills `profiles` on auth.user insert | DailyMint demo only |
| Migration `20260408144934_…sql` | DailyMint schema | DailyMint demo only |

**Nothing else.** Zero enterprise domain tables, zero edge functions, zero
storage buckets, zero domain secrets.
